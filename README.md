# EEZシミュレーター

日本の排他的経済水域(EEZ)を地図上で可視化し、**島をドラッグしたりON/OFFするとEEZがリアルタイムに再計算される**教育用Webアプリ。

「沖ノ鳥島のような数m²の岩が、国土面積を超える約40万km²のEEZを支えている」ことを、実際に触って体感できます。

**デモ: https://stnvoy.github.io/eez-simulator/**

## 主な機能

- **実データ表示**: Marine Regions World EEZ v12 による日本と近隣国のEEZ色分け表示(国境線つき)
- **シミュレーション**: 海岸線の基線点群から200海里・等距離中間線ルールでEEZを自前計算(Web Worker)
- **島ドラッグ**: 沖ノ鳥島・南鳥島・与那国島・対馬・小笠原/硫黄列島をドラッグ移動(確定と同じ高解像度で約40fps)
- **島のON/OFF**: 「その島がなかったら」の面積変化を即座に表示(沖ノ鳥島OFFで約−41万km²、小笠原OFFで約−54万km²)
- **各国のEEZ増減**: 島を動かすとどの国がどれだけ増減したかをリアルタイム表示
- **領土問題の三択**: 北方領土・竹島・尖閣諸島を「日本 / 係争中 / 相手国」で切替え、EEZがどう変わるかを試す
- **出典つきの解説コラム**: 計算方法と、各国の主張を*その国の政府自身が公表した文書*(外務省・中国国務院白書・韓国外交部・ロシア外務省・国連CLCS提出文書・PCA仲裁判断)へのリンクつきで提示
- **第121条トグル**: 沖ノ鳥島を「島(EEZあり)/岩(EEZなし)」で切替え、各国の法的立場の違いを体感
- **南沙諸島**: 既定は2016年仲裁判断(どの地形もEEZを生まない)。支配国を選ぶと「完全な効果を与えた場合」を思考実験として表示
- **沖縄トラフの参考線**: 中国が2012年にCLCSへ提出した大陸棚の外側限界10点。EEZ計算には使わない(制度が違う)
- **島を新設(サンドボックス)**: 海の任意地点に、凡例のどの国の島でも置ける
- **世界のEEZ面積ランキング**: 各国のEEZを「国土の◯倍」つきで比較(フランス18.5倍・日本11.8倍)
- **他国フォーカス**: 地図上のEEZをクリックする(またはプルダウンで選ぶ)と、その国を主役にして面積・増減を見る
- **領海(12海里)とEEZ(200海里)の同心円**: 桁違いのスケール差を可視化
- **距離測定**: 地図上の2点をクリックしてkm/海里で測る
- **もしもシナリオ**: プリセットをワンタップで再現
- **島情報カード**: マーカークリックで実面積・EEZ寄与・豆知識を表示

領有権が争われている島(択捉島・南沙諸島)は、**移動もON/OFFもできません**。領有権は条約と歴史と実効支配の問題であって島の位置の問題ではなく、動かせてしまうと「主権とは幾何学である」という誤った感覚を残すためです。これらの島で意味のある操作は「どちらの国の基点として使うか」だけなので、それだけを残しています。

## 開発

```bash
npm install
npm run dev        # 開発サーバー (http://localhost:5173)
npm test           # ユニットテスト (Vitest)
npm run build      # 本番ビルド (dist/)
npm run fetch-data # データ再生成(Marine Regions WFS / Natural Earthから取得)
```

データは `public/data/` に静的GeoJSON/JSONとしてコミット済みで、実行時に外部APIへは依存しません。

## アーキテクチャ

```
scripts/fetch-data.mjs   データ事前処理(取得→クリップ→簡略化→基線サンプリング)
src/engine/              計算エンジン(純粋関数): 球面幾何・3D kd木・グリッド分類
src/engine/worker.ts     Web Worker(差分更新の状態保持)
src/sim/controller.ts    操作フロー制御(ドラッグのフレーム間引き等)
src/store/               Zustandストア
src/components/          MapView(MapLibre GL) / SidePanel
```

EEZ計算は「グリッド各セルを最近傍基線点の国に割り当てる」方式(等距離中間線=Voronoi配分と等価)。距離は単位球面上のkd木による厳密な大圏距離で、ドラッグ中は移動の影響範囲(旧位置・新位置の200海里圏)のみ差分再計算します。

## 精度と、モデルに含まれる立場について

教育目的の簡略モデルであり、法的な境界を示すものではありません。

- 純粋な等距離中間線モデルで統一計算(現実の漁業協定線・未画定海域は再現しない)
- すべての基線点に等しい効果を与えている。現実の境界画定では小さな島の効果は減じられることが多い
- 竹島・尖閣諸島・北方領土の周辺海域は、実データ・シミュレーションのどちらでも既定は「係争中」。灰色+斜線で描き、どの国の面積にも算入しない

### 「日本のEEZ」の3つの数字

| 数え方 | 係争海域を除く | 係争海域を日本に算入 |
|---|---:|---:|
| 実データ(Marine Regions) | 4,066,513 | 4,438,264 |
| 本アプリの自前計算 | 4,492,093 | 4,833,059 |
| 日本の公称値(領海＋EEZ) | — | 4,470,000 |

アプリの既定は「係争中」です。実データ表示では 4,066,513、シミュレーションでは 4,492,093 が出ます。係争地を相手国に割り当てても日本の面積は 4,492,093 のまま変わりません(もともと算入していないため)。

Marine Regions は争いのある海域を別ポリゴンに切り出しているため、実データ表示は日本の公称値より約37万km²小さくなります。**領海を含む/含まないの違いではありません**(MRのポリゴンは領海を含む。海岸から8kmの点も内側にあることを点内外判定で確認済み)。

同じ「係争海域を除く」基準でも、自前計算はMRより 42.6万km²(10.5%) 大きくなります。内訳の主なものは:

| | 自前計算 | Marine Regions |
|---|---:|---:|
| 北方領土周辺 | 209,292 | 213,723 |
| 竹島周辺 | 62,961 | 1,624 |
| 尖閣諸島周辺 | 68,714 | 73,794 |

竹島の差が大きいのは、このアプリが竹島に完全な効果(200海里)を与えるためです。加えて日韓暫定水域(82,610 km²)を中間線で機械的に割っています。現実の境界画定では小さな島の効果は減じられるのが普通で、共同管理水域という概念もこのアプリにはありません。

### グリッドの離散化ノイズ

セルの中心が200海里以内かどうかで判定しているため、孤立した島を1セル分(約4.6km)動かすだけで面積が **約300km² 揺れます**。球面上の測地円の面積は緯度に依存しない(`A = 2πR²(1−cosθ)` = 430,893 km²)ので、これは物理ではなく画素を数えることによるノイズです。標準偏差は約120km²(日本のEEZ総面積の0.0025%)。表示している数字の下3桁はこの揺らぎの中にあります。

**中間線モデルは中立ではありません。** 東シナ海で中間線による境界画定を主張しているのは日本であり、中国と韓国は大陸棚について「自然の延長」に依拠した主張を行っています。つまりこのアプリの計算モデルは、この海域については日本の立場と一致します。この事実はアプリ内の「EEZはどう計算しているか」コラムに明記しています。

また、中国はEEZの境界線を公表していません([内閣官房](https://www.cas.go.jp/jp/ryodo/kenkyu/senkaku/chapter04_column_03.html))。そのため尖閣諸島で「中国」を選んでも、沖縄トラフまでEEZを広げるようなことはしません。存在しない主張を代弁することになるからです。切り替わるのは基線点の帰属だけです。

## 情報の時点について

**本シミュレーターは2026年7月時点で公開されていた情報をもとに作成されています。** 各国の主張、国際機関の判断、条約の解釈、参照先の文書URLは、いずれも変わりうるものです。

コラムに掲載した出典は、すべて実際に内容を確認したうえで引用しています。出典を追加・更新した際は、`src/data/columns.ts` の `SOURCES_AS_OF` も併せて更新してください。この値がアプリのフッターと各コラムの末尾に表示されます。
